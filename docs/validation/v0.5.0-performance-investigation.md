## Parallel Retrieval Investigation

Implementation:
- Parallelized PubMed, OpenAlex, Europe PMC and Semantic Scholar retrieval using asyncio.gather()
- Preserved canonical source merge order

Results

Before parallelization (warm runs):
- 6306 ms
- 7601 ms

After parallelization (warm runs):
- 3818 ms
- 2992 ms

Observed improvement:
- Approximately 40–60% reduction in warm All Sources response time.

Conclusion:
- Parallel retrieval provides significant performance improvement.
- External source retrieval remains the dominant latency component.
- PubMed remains the slowest source during cold runs.
