## Performance Investigation

### Completed

- [x] Measure All Sources search latency per source
- [x] Measure deduplication overhead
- [x] Measure sorting overhead
- [x] Measure export generation time
- [x] Compare cold-cache vs warm-cache performance
- [x] Evaluate deque-based source interleaving
- [x] Parallelize All Sources retrieval
- [x] Investigate PubMed candidate limit reduction

### Findings

- PubMed dominated cold-cache latency (~32s)
- Parallel retrieval reduced warm All Sources latency from ~6–8s to ~3–4s
- Deduplication overhead is negligible (<15 ms)
- Sorting overhead is negligible (<10 ms)
- Pagination overhead is negligible (<1 ms)
- Reducing candidate limit from 2000 to 1000 or 500 significantly reduced result volume
- Lower candidate limits produced no meaningful warm-run latency improvement
- OpenAlex and Semantic Scholar are now the dominant warm-run bottlenecks

### Conclusion

Keep:

- Parallel retrieval enabled
- Candidate limit at 2000

No further PubMed candidate-limit optimization recommended.
