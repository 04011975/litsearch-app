# LitSearch v0.5.0 Release Notes

## Summary

v0.5.0 focuses on architecture stabilization, All Sources refactoring, export parity, maintenance cleanup, and performance investigation.

## Highlights

- Centralized All Sources retrieval pipeline for UI and exports.
- Reduced duplicated sorting logic.
- Extracted paper detail routes and service logic.
- Migrated FastAPI startup/shutdown handling to lifespan.
- Modernized OpenPyXL font-copy usage.
- Improved PubMed cold-cache performance by increasing EFetch batch size from 20 to 50.
- Improved Europe PMC stability by reusing a persistent HTTP session.
- Completed OpenAlex, Semantic Scholar, All Sources caching, and export-path performance reviews.

## Performance

- Parallelized All Sources retrieval.
- Reduced PubMed cold-cache latency by approximately 46% by increasing the EFetch batch size from 20 to 50.
- Improved Europe PMC request performance through HTTP session reuse.
- Completed performance investigations for OpenAlex, Semantic Scholar, All Sources caching, and export-path optimization opportunities.

## Validation

- Full test suite: 42 passed, 7 skipped.
- Docker sanity-check passed.
- Search, source-specific exports, All Sources, detail pages, Europe PMC cursor flow, and async export jobs validated.

## Decision

v0.5.0 is considered complete. Future work moves to v0.6.0 planning.
